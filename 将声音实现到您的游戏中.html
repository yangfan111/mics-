将声音实现到您的游戏中

现阶段，您应该能够通过 Wwise 在您的游戏中播放声音了。If you haven't already set up your Wwise project with a few sounds, Events, and SoundBanks, you will need to start Wwise and do the following:

  1. 导入一个 WAV 文件。Drag and drop one from a File Explorer window to the Project Explorer's Audio tab. 这将创建一个声音。
  2. 创建一个事件。Right-click on the sound and select New Event. 给它起一个合适的名字。
  3. 创建一个 SoundBank。进入 SoundBanks 布局（F7）Click New… in the SoundBank Manager.
  4. Put your Event in the SoundBank. 为此，将 Event 从 Project Explorer 中拖放到 SoundBank 中。
  5. Click Generate Selected. 现在您就有声音可以播放了。

现在，为了将一个声音集成到您的 Unity 工程中：

  1. 打开 Wwise Picker 窗口（Window > Wwise Picker），它应该显示出您的 Events 和 SoundBanks。
  2. 将您的 SoundBank 拖放到场景中的对象上。
  3. 将您的 Event 拖放到场景中的一个对象上。
  4. 玩一下您的游戏吧！您应该能在启动时听到声音。

什么也听不到吗？想了解更多，参见 故障排除 页面。

集成更多的声音

当前 Unity 中 Wwise 的集成提供了一些 Unity 组件，这些组件足以应对一些常规的使用情况。

  • 使用 AkAmbient 来放置场景中的发声体。
  • 使用 AkBank 来加载 SoundBank。
  • 将 AkEnvironment 附加到碰撞体上，用来在指定区域内设置混响。
  • 了解其它组件。

然而，重要的是，如果想在游戏中实现更复杂的行为，需要程序员编写脚本来支持。您可以在这两节中找到更多关于 Wwise API 的信息：

  • Integrate Wwise Elements into Your Game
  • AK::SoundEngine Namespace Reference

对您 Unity 工程作出的变更

在安装过程中，对您的 Unity 工程进行了以下更改：

  • Edit > Project Settings > Audio 中的 Disable Audio 选项被选中了。内置的音频系统将因此被禁用。
  • 如果 "Create WwiseGlobal GameObject" 复选框被选中，那么在当前场景中将创建一个名为 "Wwise Global" 的游戏对象。该对象具有全局脚本 AkInitializer 和 AkTerminator 来初始化 Wwise。在初始化器脚本中，有一些可自定义的选项。The WwiseGlobal object is also given AkEditorEventPlayer, AkAudioListener and AkGameObj scripts which are used for previewing AkEvent objects in edit mode. A Rigidbody component is added such that interactions between AkEnvironment, AkRoom and AkGameObj can also be previewed.
  • Script Execution Order 被设置为加入关键的 Wwise 脚本（AkInitializer，AkGameObj，AkAudioListener，AkBank 和 AkTerminator）。 如果选中 "Add Ak Listener to Main Camera" 复选框，Unity 自带的 Audio Listener 组件便会从 Main Camera 对象中移除，而默认的 AkAudioListener.cs 脚本已经设置在 Main Camera 对象上了。
  • 设置了 Play in Background 选项，以便您可以连接 Wwise Profiler
